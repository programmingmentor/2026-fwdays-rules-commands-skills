# Wrap-up: що забираємо з собою

<v-clicks>

- Правила (Rules) як постійний контекст: короткі, перевірні, без конфліктів
- Команди (Commands) як стандартизовані запити: швидко, повторювано, “однакова форма”
- Навички (Skills) як процедури: кроки + checks + DoD.

</v-clicks>

# Фінальний DoD воркшопу (перевірка)

<v-clicks>

- У репозиторії є: rules + commands + skills (Cursor і Claude треки)
- Працює advanced workflow: згенерували PNG і додали слайд
- `npm run build` проходить
- Секрети не потрапили в git (ключі тільки в env).

</v-clicks>

# Як підтримувати систему (після воркшопу)

<v-clicks>

- Додавайте правило, якщо агент **повторює** одну й ту саму помилку
- Оновлюйте правила разом зі зміною стеку/архітектури
- Зміни rules/skills — через PR + review (як CI/CD конфіги)
- Тримайте артефакти короткими: “менше = краще”.

</v-clicks>
